# Project: Local Agreement Compliance Checker (RBI Standards)

## 1. Project Overview
Build a full-stack web application that allows users to upload legal agreements (PDF/Word), parse them (including OCR), and analyze them against predefined RBI (Reserve Bank of India) compliance clauses using a locally hosted AI model (Ollama). The system must run offline-capable logic, store data locally (SQLite), and provide detailed compliance reports with risk scoring.

## 2. Tech Stack Requirements
- **Frontend:** React (Vite) + Tailwind CSS + ShadCN UI (for modern tables, accordions, and dialogs).
- **Backend:** Python FastAPI (robust handling of files and AI logic).
- **Database:** SQLite (local file-based storage for offline access and history).
- **Document Parsing:** 
  - PDF: `pdfplumber` (for text + coordinates) + `pdf2image` + `pytesseract` (for OCR).
  - Word: `python-docx`.
- **AI Integration:** Ollama API (connect to `http://localhost:11434`). Use model `llama3` or `mistral`.
- **Report Generation:** `reportlab` or `weasyprint` (to export reports as PDF).
- **State Management:** React Query or Zustand.

## 3. Database Schema (SQLite)
Create models using SQLAlchemy:
1. **AnalysisSession:** `id`, `title`, `created_at`, `status`.
2. **Document:** `id`, `session_id`, `file_type` (Agreement, Amendment, Addendum, MOU), `upload_date`, `filename`, `effective_date`, `creation_date`.
3. **RBIClause:** `id`, `clause_text`, `predefined_meaning`, `category`.
4. **ComplianceResult:** `id`, `document_id`, `rbi_clause_id`, `compliance_status` (Compliant/Non-Compliant/Review), `risk_score` (0-100), `agreement_reference` (e.g., "Clause 3.1, Line 10"), `ai_understanding_agreement`, `ai_understanding_rbi`.

## 4. Core Backend Logic & Workflows

### A. Document Parsing & OCR
- Create an endpoint `/upload`.
- Detect file type (PDF/DOCX).
- **Text Extraction:** Extract raw text AND try to preserve location metadata (Page Number, approximate Line/Clause number) for the "Reference Point" requirement.
- **OCR:** If text extraction returns low confidence or empty strings, trigger OCR pipeline using Tesseract.
   - **Date Extraction:** Automatically search for "Date of Agreement" and "Effective Date". Regex pattern matching is required. If missing, mark as 'NA' and allow manual edit in UI.

### B. AI Analysis Pipeline (Ollama)
Create a service module `ai_analyzer.py`. All prompts must force JSON output.
1. **Step 1 (Ingestion):** Send Agreement Text to Ollama. 
   - *Prompt:* "Extract all distinct clauses. For each, provide: Clause ID (e.g., 3.1), Text, and a simplified meaning."
2. **Step 2 (RBI Context):** Load predefined RBI clauses from DB. Send them to Ollama to generate "Predefined Meaning" embeddings/summaries if not already stored.
3. **Step 3 (Compliance Check):** Send Agreement Clauses + RBI Clauses to Ollama.
   - *Prompt:* "Compare Agreement Clause X against RBI Clause Y. Determine Compliance Status, Risk Score (0-100), and cite the specific Reference Point in the agreement."
4. **Step 4 (Incremental Logic):** 
   - If **New Agreement**: Run full comparison against all RBI clauses.
   - If **Amendment/Addendum**: Compare *only* new/changed text against RBI clauses. Mark existing clauses as "Previously Covered" and new ones as "New Clause".
   - If **MOU**: Analyze understanding and check against relevant RBI clauses, flagging if it supersedes the main agreement.

### C. API Endpoints
- `POST /api/analysis/create`: Start a new session (Title, Date).
- `POST /api/document/upload`: Handle file upload, parsing, and trigger AI analysis.
- `GET /api/analysis/{id}`: Fetch session details, documents, and compliance results.
- `GET /api/report/{id}`: Generate PDF report.
- `DELETE /api/document/{id}`: Remove a document from the session.

## 6. Specific Logic Constraints
1. **Reference Pointing:** The parser must attempt to map text back to "Clause X, Line Y". If exact line number is impossible via code, instruct the AI to estimate the reference based on context and label it "Approximate Reference".
2. **Date Handling:** Do not validate if Effective Date is before Creation Date. Allow pre-dated logic. Default to "NA" if not found, allow manual edit.
3. **One-Liner Guidelines:** If the agreement says "We follow all RBI guidelines", the AI must interpret this in Table 2 as "General Compliance Statement" and flag in Table 3 that specific clause mapping is required for detailed verification.
4. **Local First:** The code must be configured to connect to `http://localhost:11434` for Ollama. Include a `.env` example for API configuration.
5. **Security:** No data should be sent to external cloud APIs (except the local Ollama instance). All files stored in a local `/uploads` directory.
